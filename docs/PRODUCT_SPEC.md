# MiLuStudio Product Spec

更新时间：2026-05-12

## 产品定位

MiLuStudio 是 Windows 原生 AI 漫剧 Agent 产品。

用户输入故事、小说片段或创作要求。系统自动完成剧本、角色、风格、分镜、图片、视频、配音、字幕、剪辑、质检和导出。用户只在关键节点做确认、选择和轻量修改。

## 第一版 MVP

输入：

- 500 到 2000 字中文故事或小说片段。
- 目标时长：30 到 60 秒。
- 默认画幅：9:16。
- 模式：极速模式 / 导演模式。

输出：

- 脚本卡。
- 角色卡。
- 风格卡。
- 分镜表。
- 图片素材。
- 视频片段。
- 字幕文件。
- 最终 MP4。
- 工程文件或素材包。

## 普通用户界面

第一版只保留四块：

1. 对话输入区。
2. 任务进度流。
3. 中间结果卡片。
4. 最终交付区。

默认停顿点：

- 脚本确认。
- 角色确认。
- 风格确认。
- 分镜确认。
- 图片结果确认。
- 视频结果确认。
- 最终质检确认。

极速模式可以跳过中间确认，但必须在成本明显增加前提示用户。

## 视觉基调

- 整体参考 XiaoLouAI 的浅色、蓝紫、简洁工作台气质。
- 不复制 XiaoLouAI 源码或视觉资产，只参考颜色、留白、轻边框和操作台感。
- 前端应优先呈现“AI 漫剧生产控制台”，避免营销页、重装饰和复杂画布。
- 主要操作色使用蓝紫到紫色系，背景保持清爽，卡片阴影和边框保持克制。

## 品牌资产

- 项目 logo 必须内置在项目目录中。
- Web 前端当前使用 `apps\web\public\brand\logo.png`。
- 页面品牌标识和 favicon 统一引用项目内路径 `/brand/logo.png`。
- 不允许在代码、HTML 或运行时配置中引用下载目录、用户目录或其他项目外部 logo 路径。

## 第一版不做

- 不做通用 AI 助手。
- 不做无限画布。
- 不做复杂时间线。
- 不做节点拖拽编排。
- 不做公共 Skills 市场。
- 不做第三方插件安装。
- 不做多用户协同。
- 不做云端 SaaS 管理后台。
- 不把 Linux / Docker 作为生产必需。

## 技术边界

- 前端：React / Vite / TypeScript。
- 后端：.NET 8 / ASP.NET Core。
- 数据库：SQLite 本地文件数据库。
- Worker：Windows Worker。
- Sidecar：Python Skills Runtime。
- 桌面：Electron / electron-builder / NSIS。

前端只展示、确认和编辑，不直接访问数据库、文件系统、模型 SDK、Python 脚本或 FFmpeg。

## 目录组织

- 根目录保持简洁。
- 长期文档和交接记录放入 `docs\`。
- 前端按路由或功能域聚合。
- 后端按业务能力和层边界聚合。
- Production Skills 按技能目录聚合。

## 安全和本地数据

- API Key 只存在本机。
- 不提交 `.env`、真实素材、输出视频、日志或服务账号。
- 本地运行数据、上传素材和生成结果放入 `storage\`、`uploads\`、`outputs\` 或用户明确选择的位置。
- 依赖、缓存、运行文件和生成数据优先约束在 `D:\code\MiLuStudio` 内。
